# open-async-task 异步任务
## 设计思想
1. 用户使用的时候传入的参数最少
2. 分段任务设计：
   3. 第一步：根据用户传入的少量参数查询出需要的对象。然后丢到第二个异步任务中
   4. 第二步：根据上一个异步任务的结果继续后续的执行
5. 同步或异步
   6. 同步：两个任务都完成了返回结果，其中一个任务报错则报错
   7. 异步：同步根据用户传入的信息查询后一个任务需要的对象，丢到后面的任务池中返回结果，不等待后续的异步任务执行结果。

## package解释
- calculate：核心异步任务的逻辑
- calculate.business：业务代码
